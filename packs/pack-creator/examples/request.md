# Example Request

Use `Pack Creator` when you need to create a new reusable pack from a domain brief and one realistic example.

Example:

> Create a `compliance-review` pack for policy-heavy release checks.
> Intended users: runtime operators and reviewers.
> Artifact types: review report, issue list, approval recommendation.
> Constraints: must mark confirmed vs inferred findings and preserve source evidence.
> Validation example: review a browser transport release candidate before merge.
