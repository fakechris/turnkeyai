# Pack Creator Intake

## Inputs

- target domain name and short summary
- intended users or operators of the future pack
- artifact types the pack should own
- hard constraints, policy boundaries, and non-goals
- one concrete example request that the final pack must be able to handle

## Checklist

1. Restate the domain boundary in one sentence and name what is explicitly out of scope.
2. Decide the pack id, display name, and one-line summary.
3. Define the first capability map and workflow map.
4. List the minimum reference sections the pack will need to stay reusable.
5. Choose one validation example to prove the generated pack works beyond empty placeholders.

## Output

- normalized pack brief
- proposed manifest fields
- capability map
- workflow map
- validation example
- open questions
