# Media Pack Pattern

`media-pack` is the reference example for this skill.

Use it to see how a generated skeleton should be refined into a real domain pack:

- the pack boundary names image, video, audio, and adaptation workflows
- the intake recipe collects source assets, rights, channels, and delivery formats
- the execution recipe records provenance, tool choices, and packaging
- the domain model names artifact types, constraint types, and review rubrics
- the example request is concrete enough to test the pack without hidden context

When creating a new pack, aim for the same level of specificity, but keep the domain language specific to the new pack.
